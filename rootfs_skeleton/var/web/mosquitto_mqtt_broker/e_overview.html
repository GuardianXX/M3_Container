Mosquitto MQTT-Broker

This is a demonstration container running in the SmartBox of MRX routers. It is provided by INSYS icom on the base of existing 3rd party software especially the MQTT broker Mosquitto. MQTT is one of the leading protocols for the IoT (Internet of Things) as its publisher-subscriber architecture realises the necessary information flow without polling in slim and reliable way. Within the MQTT architecture the broker accepts topics with values from publishes and forwards them to subscribers registered for these.

Publisher and subscriber need to establish TCP/IP connections to the broker. The routers networking capabilities (e.g. OpenVPN over INSYS Connectivity Service) can support the network accessibility. To operate this showcase you need
  • an MQTT publisher (e.g. Monitoring App on INSYS OS routers or demo container Modbus MQTT gateway for MRX SmartBox) as a data source
  • an MQTT subscriber application running on a PC or smartphone (see recommendations on HiveMQ/Clients) to display the values

Configuration

  • The bind address is used to restrict the access and may remain empty for demonstration purposes.
  • The TCP port is the target port for connection from publishers and subscribers. Default ports are 1883 and 8883 (for TLS).
  • The IP address of the container itself as the target address for connection from publishers and subscribers is managed in the router's menu Administration/Container
Beware: Changes of the configuration take effect only after clicking on activate settings in the headline.
